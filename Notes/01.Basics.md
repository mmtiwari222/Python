## 1. Introduction to Python

Python is a **high-level, interpreted, general-purpose programming language** known for its simplicity and readability.

```python
print("Hello, World!")  # Your first Python program
```

> 💡 **Interview Tip:** Python is **interpreted** (not compiled) — code runs line by line via the Python Virtual Machine (PVM). Easier to debug, but slightly slower than compiled languages like C++.

## 5. Python Output — `print()`

```python
print("Hello, World!")              # Basic output
print("A", "B", "C")               # → A B C
print("A", "B", sep="-")           # → A-B
print("Hello", end=" ")            # No newline at end
print("World")                     # → Hello World (same line)
```

| Parameter | Default | Purpose |
|-----------|---------|---------|
| `sep` | `" "` (space) | Separator between multiple values |
| `end` | `"\n"` (newline) | What to print at the end |

> 💡 **Interview Tip:** `print("a", "b", sep=",", end="!")` → `a,b!` — common MCQ trap in coding rounds.

## 6. Data Types in Python

### Primitive (Scalar) Types

| Type | Example | Notes |
|------|---------|-------|
| `int` | `42`, `-7` | Whole numbers, unlimited precision |
| `float` | `3.14`, `2.0` | Decimal numbers (64-bit) |
| `str` | `"hello"` | Immutable sequence of characters |
| `bool` | `True`, `False` | Subclass of `int` — True=1, False=0 |

### Collection Types

| Type | Example | Ordered | Mutable | Duplicates |
|------|---------|:-------:|:-------:|:----------:|

## 7. `type()` Function

Returns the **data type (class)** of any object.

```python
type(42)        # <class 'int'>
type(3.14)      # <class 'float'>
type("hello")   # <class 'str'>
type(True)      # <class 'bool'>
type([1, 2])    # <class 'list'>
type(None)      # <class 'NoneType'>
```

> 💡 **Interview Tip:** Everything in Python is an **object** — even functions and classes. `type()` inspects the class of any object at runtime.

## 8. Variables in Python

A variable is a **name (label) bound to an object in memory**.

```python
x = 10        # x points to integer object 10
x = "hello"   # now x points to string — no error! (dynamic typing)
```

### 8.1 Dynamic vs Static Typing

| Feature | Dynamic Typing (Python) | Static Typing (Java/C++) |
|---------|------------------------|--------------------------|
| Type check | At **runtime** | At **compile time** |
| Declaration | Not needed | Must declare type |
| Flexibility | High | Low |

### 8.2 Dynamic vs Static Binding

| Concept | Definition |
|---------|------------|
| **Dynamic Binding** | Variable name bound to object at **runtime**; can be rebound anytime |
| **Static Binding** | Name-to-type binding fixed at **compile time** |

Python uses **dynamic binding** — variables are references/pointers to objects, not fixed containers.

> 💡 **Interview Tip:** Python variables are like **sticky notes on objects**, not boxes. `x = 10` → stick label `x` on object `10`. `x = "hi"` → move the label.

## 10. Keywords & Identifiers

### Keywords
Reserved words with special meaning — **cannot** be used as variable names.

```python
import keyword
print(keyword.kwlist)   # Lists all Python keywords
```

Common keywords: `if`, `else`, `elif`, `for`, `while`, `def`, `class`, `import`, `return`, `True`, `False`, `None`, `and`, `or`, `not`, `in`, `is`, `lambda`, `pass`, `break`, `continue`, `try`, `except`

### Identifiers — Naming Rules

| Rule | Valid | Invalid |
|------|-------|---------|
| Letters, digits, underscore only | `my_var`, `var1` | `my-var`, `var!` |
| Cannot start with digit | `_name`, `name1` | `1name` |
| Case-sensitive | `Name` ≠ `name` | — |
| Cannot be a keyword | `total` | `for`, `class` |

> 💡 **Interview Tip:**
> - `_variable` → private by convention
> - `__variable` → name mangling in classes
> - `__variable__` → dunder/magic methods

## 11. User Input

```python
name = input("Enter your name: ")   # Always returns STRING
age = int(input("Enter age: "))      # Must convert explicitly
```

### ❓ Why does Python always take input as string?

`input()` reads from **stdin** as a raw text stream. The user types characters — Python cannot know if `"42"` is meant to be an int, float, or string without explicit instruction. So Python plays it safe → always returns `str`.

> 💡 **Interview Tip:** Forgetting to convert `input()` is a classic bug. Always use `int(input())` or `float(input())` when you need numbers.

## 12. Type Conversion

### 12.1 Implicit Type Conversion (Coercion — done by Python)

Python automatically converts to prevent data loss — always **upcasts** (smaller → larger type).

```python
x = 5       # int
y = 2.0     # float
z = x + y   # Python auto-converts int → float
# z = 7.0  (float)
```

> ⚠️ Only works between **compatible types** (int ↔ float). `str + int` → `TypeError`.

### 12.2 Explicit Type Conversion (Type Casting — done by programmer)

| Function | What it does |
|----------|--------------|
| `int(x)` | Truncates float (no rounding). `int("42")` ✅ `int("42.5")` ❌ |
| `float(x)` | Converts to float. `float("3.14")` → `3.14`. `float(True)` → `1.0` |
| `str(x)` | Converts anything to string. `str(42)` → `"42"` |
| `bool(x)` | **Falsy** → `False`: `0`, `0.0`, `""`, `[]`, `{}`, `None`. Everything else → `True` |

```python
int(3.9)      # → 3   (truncation, NOT rounding ⚠️)
int(True)     # → 1
int(False)    # → 0
bool(0)       # → False
bool("")      # → False
bool("0")     # → True  ← non-empty string! Common trap ⚠️
```

> 💡 **Interview Tips:**
> - `int(3.9)` → `3`, **NOT** `4` — Python **truncates**, doesn't round
> - `bool("0")` → `True` — non-empty string is always truthy

## 13. Type Casting vs Type Coercion

| Concept | Definition | Who does it? |
|---------|------------|-------------|
| **Type Casting** | Explicit conversion using `int()`, `str()`, etc. | Programmer |
| **Type Coercion** | Implicit/automatic conversion | Python interpreter |

In Python, coercion is **limited** (only numeric: int ↔ float). Other conversions require explicit casting.

> 💡 **Interview Tip:** JavaScript aggressively coerces (`"5" + 3 = "53"`). Python is **strict** — incompatible types raise `TypeError` instead of silently converting.

## 14. Literals

Literals are **fixed values written directly in code**.

| Type | Examples |
|------|---------|
| Integer | `42`, `0`, `-7`, `0b1010` (binary), `0o17` (octal), `0xFF` (hex) |
| Float | `3.14`, `2.0`, `1.5e3` (scientific = 1500.0) |
| String | `"hello"`, `'world'`, `"""multiline"""` |
| Boolean | `True`, `False` |
| None | `None` |
| Complex | `3+4j` |
| List | `[1, 2, 3]` |
| Tuple | `(1, 2, 3)` |
| Set | `{1, 2, 3}` |
| Dict | `{"key": "value"}` |

> 💡 **Interview Tip:** `None` is Python's null value — singleton of type `NoneType`. Always check `x is None` (not `x == None`) — `is` checks **object identity**.
