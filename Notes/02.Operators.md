## What is an Operator?

An **operator** is a symbol that tells Python to perform a specific operation on values called **operands**.

```python
x = 10 + 5   # + is operator | 10 and 5 are operands | 15 is the result
```

Python has **8 types** of operators:

| # | Type | Purpose |
|---|------|---------|
| 1 | Arithmetic | Math calculations |
| 2 | Assignment | Assign values to variables |
| 3 | Comparison | Compare values → returns bool |
| 4 | Logical | Combine boolean expressions |
| 5 | Bitwise | Operate on binary bits |
| 6 | Identity | Check same memory object |
| 7 | Membership | Check existence in sequence |
| 8 | Ternary | One-line if-else |

## 1. Arithmetic Operators

| Operator | Name | Example | Result |
|----------|------|---------|--------|
| `+` | Addition | `10 + 3` | `13` |
| `-` | Subtraction | `10 - 3` | `7` |
| `*` | Multiplication | `10 * 3` | `30` |
| `/` | Division | `10 / 3` | `3.333...` (always float) |
| `//` | Floor Division | `10 // 3` | `3` |
| `%` | Modulus | `10 % 3` | `1` |
| `**` | Exponentiation | `2 ** 3` | `8` |

```python
print(10 / 3)    # 3.3333333333333335  ← always float!
print(10 // 3)   # 3
print(-7 // 2)   # -4  ← floors toward -∞, NOT toward zero! ⚠️
print(10 % 3)    # 1
print(-7 % 2)    # 1   ← sign follows divisor in Python ⚠️
print(2 ** 10)   # 1024
print(2**3**2)   # 512 ← right-associative: 3**2=9 first ⚠️
```

> 💡 **Interview Tips:**
> - `/` **always** returns `float` in Python 3 — even `4/2 = 2.0`
> - `//` floors toward **negative infinity** — `-7 // 2 = -4`, not `-3`
> - `%` result has **same sign as the divisor** — `-7 % 2 = 1`
> - `**` is **right-associative** — `2 ** 3 ** 2 = 2 ** 9 = 512`

## 2. Assignment Operators

| Operator | Equivalent To | Example |
|----------|--------------|---------|
| `=` | — | `x = 10` |
| `+=` | `x = x + n` | `x += 5` |
| `-=` | `x = x - n` | `x -= 3` |
| `*=` | `x = x * n` | `x *= 2` |
| `/=` | `x = x / n` | `x /= 4` → float |
| `//=` | `x = x // n` | `x //= 3` |
| `%=` | `x = x % n` | `x %= 3` |
| `**=` | `x = x ** n` | `x **= 2` |
| `:=` | Walrus — assign + return | `(n := 10)` |

```python
# Walrus operator := (Python 3.8+)
if (n := len("hello")) > 3:
    print(f"Long string: {n} chars")   # n = 5

# Common use in while loop
while chunk := file.read(1024):
    process(chunk)
```

> 💡 **Interview Tips:**
> - Python has **no `++` or `--`** — use `x += 1`
> - `:=` (walrus operator) is useful in `while` loops and comprehensions
> - `/=` always gives a `float` result

## 3. Comparison (Relational) Operators

Always return `True` or `False`.

| Operator | Meaning | Example | Result |
|----------|---------|---------|--------|
| `==` | Equal to | `5 == 5` | `True` |
| `!=` | Not equal | `5 != 3` | `True` |
| `>` | Greater than | `5 > 3` | `True` |
| `<` | Less than | `3 < 5` | `True` |
| `>=` | Greater or equal | `5 >= 5` | `True` |
| `<=` | Less or equal | `3 <= 5` | `True` |

```python
print(5 == 5.0)      # True  ← numeric value comparison across types
print("5" == 5)      # False ← different types
print(1 < 2 < 3)     # True  ← chained comparisons! ✅
print(1 < 2 > 0)     # True  ← (1<2) and (2>0)
```

> 💡 **Interview Tips:**
> - `==` checks **value equality**; `is` checks **memory identity** — don't confuse them
> - Python supports **chained comparisons**: `0 < x < 10` works natively
> - `5 == 5.0` → `True` — Python compares numeric values across types

## 4. Logical Operators

| Operator | Meaning | Example | Result |
|----------|---------|---------|--------|
| `and` | Both must be True | `True and False` | `False` |
| `or` | At least one True | `True or False` | `True` |
| `not` | Reverses bool | `not True` | `False` |

### 🔑 Short-Circuit Evaluation

```python
# and → returns first FALSY value, or last value if all truthy
print(0 and 5)       # 0    ← short-circuits at 0
print(3 and 5)       # 5    ← both truthy, returns last
print("" and "hi")   # ""   ← short-circuits

# or → returns first TRUTHY value, or last value if all falsy
print(0 or 5)        # 5    ← 0 is falsy, returns 5
print(3 or 5)        # 3    ← short-circuits at first truthy
print("" or "hi")    # "hi"
print(0 or "" or []) # []   ← all falsy, returns last

# not always returns True or False
print(not 0)         # True
print(not "hello")   # False
```

### Truth Table

| A | B | A and B | A or B | not A |
|:---:|:---:|:-------:|:------:|:-----:|
| True | True | True | True | False |
| True | False | False | True | False |
| False | True | False | True | True |
| False | False | False | False | True |

> 💡 **Interview Tips:**
> - `and`/`or` return **operands**, not always `True`/`False`
> - `and` → first falsy or last value | `or` → first truthy or last value
> - Common pattern: `value = user_input or "default"`
> - Short-circuit avoids unnecessary evaluation — performance benefit

## 5. Bitwise Operators

| Operator | Name | Example | Result | Binary |
|----------|------|---------|--------|--------|
| `&` | AND | `5 & 3` | `1` | `101 & 011 = 001` |
| `\|` | OR | `5 \| 3` | `7` | `101 \| 011 = 111` |
| `^` | XOR | `5 ^ 3` | `6` | `101 ^ 011 = 110` |
| `~` | NOT | `~5` | `-6` | inverts all bits |
| `<<` | Left Shift | `5 << 1` | `10` | `101 → 1010` |
| `>>` | Right Shift | `5 >> 1` | `2` | `101 → 010` |

```python
print(5 & 3)    # 1
print(5 | 3)    # 7
print(5 ^ 3)    # 6
print(~5)       # -6  ← formula: ~n = -(n+1) ⚠️
print(~0)       # -1
print(5 << 1)   # 10  ← multiply by 2^1
print(5 << 3)   # 40  ← multiply by 2^3
print(20 >> 2)  # 5   ← floor divide by 2^2
```

> 💡 **Interview Tips:**
> - `~n = -(n+1)` always — so `~0 = -1`, `~5 = -6`
> - Left shift `<< n` = multiply by `2ⁿ`
> - Right shift `>> n` = floor divide by `2ⁿ`

## 6. Identity Operators

| Operator | Meaning |
|----------|---------|
| `is` | Same object (identity check) |
| `is not` | Different objects |

```python
a = [1, 2, 3]
b = [1, 2, 3]
c = a

print(a == b)     # True  ← same value
print(a is b)     # False ← different objects in memory ⚠️
print(a is c)     # True  ← c points to same object as a

# Small integer caching: Python caches -5 to 256
x = 100; y = 100
print(x is y)     # True  ← cached! ⚠️

x = 1000; y = 1000
print(x is y)     # False ← not cached

# None check — ALWAYS use 'is'
print(x is None)   # ✅ correct
print(x == None)   # ⚠️ bad practice
```

> 💡 **Interview Tips:**
> - `==` → value equality | `is` → same memory address
> - Python **caches integers -5 to 256** — `is` on small ints can mislead
> - **Always** use `is None` — never `== None`

## 7. Membership Operators

| Operator | Meaning | Example | Result |
|----------|---------|---------|--------|
| `in` | Value exists | `3 in [1,2,3]` | `True` |
| `not in` | Value absent | `5 not in [1,2,3]` | `True` |

```python
print(3 in [1, 2, 3])           # True
print("a" in "apple")           # True  ← substring check
print("ell" in "hello")         # True
print("key" in {"key": "val"})  # True  ← checks KEYS only ⚠️
print("val" in {"key": "val"})  # False ← NOT values!
print("val" in {"key":"val"}.values())  # True ← correct way
```

> 💡 **Interview Tips:**
> - `in` on dict checks **keys only** — classic trap!
> - `in` on set is **O(1)** vs list O(n)
> - `in` on string = substring check

## 8. Ternary (Conditional) Operator

**Syntax:** `value_if_true if condition else value_if_false`

```python
# Traditional
age = 20
if age >= 18:
    status = "Adult"
else:
    status = "Minor"

# Ternary equivalent
status = "Adult" if age >= 18 else "Minor"

# Nested (use sparingly)
marks = 75
grade = "A" if marks >= 90 else "B" if marks >= 75 else "C"
print(grade)   # B
```

> 💡 **Interview Tips:**
> - Python: `X if CONDITION else Y` — different from C/Java's `condition ? x : y`
> - Ternary is an **expression** — it returns a value
> - Avoid deeply nested ternaries

## Operator Precedence

Higher = evaluated first. **When in doubt, use `()`!**

| Priority | Operator(s) | Description |
|:--------:|-------------|-------------|
| 1 ↑ | `()` | Parentheses |
| 2 | `**` | Exponentiation (right-associative) |
| 3 | `+x`, `-x`, `~x` | Unary |
| 4 | `*`, `/`, `//`, `%` | Multiplication / Division |
| 5 | `+`, `-` | Addition / Subtraction |
| 6 | `<<`, `>>` | Bitwise shifts |
| 7 | `&` | Bitwise AND |
| 8 | `^` | Bitwise XOR |
| 9 | `\|` | Bitwise OR |
| 10 | `==`, `!=`, `>`, `<`, `>=`, `<=`, `is`, `is not`, `in`, `not in` | Comparisons |
| 11 | `not` | Logical NOT |
| 12 | `and` | Logical AND |
| 13 ↓ | `or` | Logical OR |

```python
print(2 + 3 * 4)               # 14   ← * before +
print((2 + 3) * 4)             # 20   ← () first
print(2 ** 3 ** 2)             # 512  ← right-associative
print(not 1 + 1)               # False ← 1+1=2, not 2=False
print(True or False and False) # True ← and before or
```
